What is Linux?

Linux is an open-source family of Unix-like operating systems based on the Linux kernel. It is widely used for servers, desktops, embedded devices, and mobile devices. Linux is developed collaboratively and distributed under the GNU General Public License (GPL).

Overview

Linux is both a kernel (core of the OS) and a platform ecosystem (kernel + utilities + applications). The typical Linux-based OS is often called a "distribution" (or distro) — e.g., Ubuntu, Fedora, Debian.

  • Open-source: Source code is available for inspection and modification.
  • Stable & secure: Used by many servers and cloud providers.
  • Flexible: Runs on many CPU architectures and devices.
  • Customizable: Users can compose a system from packages they choose.

History (short)

Linux began in 1991 when Linus Torvalds released the initial Linux kernel. GNU tools and utilities, created by the Free Software Foundation, combined with the kernel to form fully functioning systems. The collaborative open-source model produced a rich ecosystem of distributions and projects.

The Linux Kernel

The kernel handles hardware, memory, processes, drivers, networking and system calls. It is monolithic (but modular) — many drivers and modules can be loaded/unloaded at runtime.

Processes & Scheduling
Preemptive multitasking, scheduling policies (CFS), namespaces and cgroups for isolation.
Memory Management
Virtual memory, page cache, swap, and kernel memory allocators.
Device Drivers
Support for block, character, network devices, USB, GPUs, etc.

Popular Distributions

Different distros target different users and use-cases:

  • Ubuntu: User-friendly desktop/server, great community and tutorials.
  • Debian: Stable and conservative; basis for many distros (including Ubuntu).
  • Fedora: Cutting-edge features, upstream-focused (Red Hat sponsored).
  • CentOS / Rocky / AlmaLinux: Enterprise-focused (binary-compatible with RHEL variants).
  • Arch Linux: Minimal, rolling-release, great for learning and customization.
  • openSUSE: Tumbleweed (rolling) & Leap (stable); YaST config tool.

Essential Linux Commands

Copy/paste these in a terminal to try them (don't run unknown commands as root):

# show current directory
pwd

# list files (long form, human readable)
ls -lah

# change directory
cd /path/to/folder

# view file content
cat file.txt
less file.txt

# copy / move / remove
cp source dest
mv source dest
rm file.txt

# check running processes
ps aux | less
top
htop   # if installed

# system info
uname -a
lsblk
df -h
free -h

# package manager examples
sudo apt update && sudo apt upgrade   # Debian/Ubuntu
sudo dnf update                       # Fedora
sudo pacman -Syu                      # Arch

Package Managers & Software

Each distro ecosystem has its preferred package manager:

  • APT — Debian/Ubuntu: apt install package
  • DNF / YUM — Fedora/RHEL/CentOS: dnf install package
  • PACMAN — Arch: pacman -S package
  • Zypper — openSUSE: zypper install package
  • Flatpak / Snap / AppImage — distro-agnostic packaging options for desktop apps

Tips for Learning & Using Linux

  1. Use a virtual machine (VirtualBox) or a live USB to try distros safely.
  2. Learn the shell (bash, zsh) and practice common commands daily.
  3. Read logs in /var/log when troubleshooting.
  4. Use version control (git) for your configs (dotfiles).
  5. Back up before making big system changes.
  6. Search documentation: man command and online distro docs.
